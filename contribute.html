🤝 HOW TO HELP

You don’t need to be a programmer, a lawyer, or a journalist.
If you can read, you can help. Here’s how.

💡 Why this project needs people

The U.S. Department of Justice released over 900,000 documents — more than 3 million pages — related to the Jeffrey Epstein case. This site has already extracted, indexed, and made every page searchable.

But the hard part isn’t the technology — it’s the reading. 3 million pages of depositions, flight logs, financial records, and FBI reports can’t be analyzed by one person. Thousands of people are independently combing through these files right now — all starting from scratch, all duplicating each other’s work.

This site exists so nobody has to start from scratch. It’s a shared foundation. When you find something, it gets added here. When the next person looks, they pick up where you left off instead of re-reading the same 400 pages.

📖 Ways to help that require zero technical skill

Seriously — none. These are the most valuable contributions we can get:

  • Read the documents and report what you find. Pick any dataset, any topic, any name — and read. If you spot a connection, a discrepancy, or something that seems important, report it.
  • Check existing profiles for accuracy. We have 148 profiles of people connected to the case. If you know something is wrong, incomplete, or missing a source, let us know.
  • Verify timeline events. Our case timeline tracks key events. If a date is wrong or an event is missing, that’s valuable information.
  • Identify people in photos. Our gallery contains hundreds of photos from the files. Some people in them haven’t been identified yet.
  • Cross-reference names. If you see a name in a deposition that also appears in a flight log — that’s a connection. Report it.
  • Spread the word. Share this site with anyone who cares about accountability. The more eyes on these documents, the harder they are to bury.

📢 How to report what you find

We use a free tool called GitHub to organize this project. You don’t need to understand how it works — think of it like a shared Google Doc where anyone can suggest changes and have a conversation.

Here’s how to share something you’ve found:

1 Create a free GitHub account

Go to github.com/join and sign up. It takes 30 seconds. You just need an email address.

2 Go to our Discussions page

This is where you can share findings, ask questions, or start a conversation. Think of it like a message board for this project.

Open Discussions →

3 Post what you found

Click “New discussion”, give it a title (like “Found connection between X and Y in DataSet 4”), and describe what you found. Include the file name or page number if you can. That’s it.

Or: File an Issue

If you found a mistake on the site — a broken link, a wrong date, a misspelled name — you can file an “Issue”. An issue is just a bug report. Click the button below, describe the problem, and we’ll fix it.

Report an Issue →

💬 GitHub jargon — explained in plain English

If you see these words and get confused, here’s what they mean:

Repository (“repo”)
A project folder. Our repo contains all the code and content for this site. Think of it as the master copy.
Issue
A bug report or suggestion. Like raising your hand to say “something’s broken” or “I have an idea.”
Discussion
An open conversation thread. Like a forum post where anyone can reply.
Fork
Your own personal copy of the project. You can make changes to your copy without affecting the main one.
Pull Request (“PR”)
When you’ve made changes to your copy and want to suggest them back to the main project. Like saying “here’s what I changed — want to add this?”
Commit
A saved change. Every time someone edits a file and saves it to the project, that’s a commit. It’s tracked so nothing gets lost.
Branch
A separate workspace where you can make changes without affecting the main site until you’re ready.
Merge
Combining changes from one branch into the main project. This is how suggestions get accepted and go live.
Markdown (.md files)
A simple text format used for content on this site. It’s just text with a few formatting symbols like ** for bold and # for headings. Easier than Word.

⚙️ If you’re comfortable with technology

If you know your way around a computer (even without coding experience), you can contribute more directly:

  • Edit content directly on GitHub. Every page on this site is a text file. You can click “Edit” on any file in our GitHub repo, make changes in your browser, and submit a Pull Request. GitHub will walk you through it.
  • Improve a cast profile. Profiles live in content/cast/pages/. Open one, add missing information with sources, and submit.
  • Add a timeline event. Timeline files live in content/timeline/. Copy the format from an existing entry and add a new one.
  • Write an investigation summary. If you’ve done deep research on a topic, write it up and submit it.

💻 For developers and data people

If you write code, there’s no shortage of work:

  • Fork the repo and run the site locally — it’s static HTML/CSS/JS, no build step needed.
  • Improve search functionality. The search indexes 59,000+ artifacts. It could be faster, smarter, or support better filters.
  • Build new visualizations. Network graphs, financial flow charts, geo-mapped flight routes — the data is all there.
  • Write scrapers or parsers to extract more structured data from the raw documents.
  • Help with accessibility. Screen reader support, keyboard navigation, mobile responsiveness — all welcome.

🌟 Want to help run this project?

We’re actively looking for people who want to become co-maintainers — people who help review contributions, keep the site accurate, and share the responsibility of running it. You don’t need to be a developer. If you’re organized, detail-oriented, and care about this work, we want to hear from you.

Start a conversation in Discussions or open an Issue titled “I want to help maintain this project”.
